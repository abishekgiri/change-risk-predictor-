# Phase 2 Executive Clarity Script

Use this script for `3-5` non-technical demo sessions to validate whether Phase 2 actually explains the product without narration.

## Goal

Confirm that a new viewer can understand the product in under `5 minutes`, especially:

- what the product does
- why the company would pay for it
- what the most urgent risks are
- what action should happen next

## Setup

- Use the current `/overview` page as the first screen.
- Do not explain the dashboard before the participant reacts.
- Let the participant read in silence for `30-60` seconds.

## Ask Only These Questions First

1. `What does this product do?`
2. `Why would a company pay for this?`

## Pass Signals

Count the session as a clarity pass if the participant answers with ideas like:

- `It prevents risky releases`
- `It enforces rules before deployment`
- `It shows what's wrong and how to fix it`
- `It helps a company avoid failed releases or audit problems`

## Fail Signals

Count the session as a clarity fail if the participant answers with ideas like:

- `It connects to Jira`
- `It tracks workflows`
- `It monitors releases`
- `It looks like an engineering dashboard`

## What To Watch

Write down exact phrases when the participant:

- pauses or hesitates on the top risk cards
- misunderstands a risk or action
- ignores the recommended action
- reads the page as monitoring instead of control

## Fix Rules

After each batch, change only one category at a time:

- unclear wording
- weak action language
- non-obvious urgency

Do not add more metrics or more surfaces before the narrative is clear.
